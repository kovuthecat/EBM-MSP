## T-002 — JSON Schema du nœud de décision

### Objectif

Écrire `schema/noeud.schema.json` (JSON Schema) décrivant un nœud conforme au brief §5.1, `domaine` requis.

### Lire / Modifier

- Créer : `schema/noeud.schema.json`. Créer les types TS correspondants
  `src/features/decision/content/node.types.ts` (miroir du schéma : `Noeud`, `Option`, `CritereEntree`,
  `Source`, `Meta`, `NiveauPreuve = 'eleve'|'modere'|'faible'|'tres_faible'`).

### Étapes

1. Modéliser : `id`, `domaine` (**required**), `titre`, `population_cible`, `criteres_entree[]`
   (`{nom, type: 'nombre'|'bool'|'enum', valeurs?: string[]}`), `options[]`
   (`{intitule, avantages[], inconvenients[], effet_attendu, niveau_preuve, conditions[], contre_indications[]}`),
   `argumentaire`, `sources` (`references_primaires[{titre,annee,lien,type_critere}]`,
   `medicalement_geek{synthese,lien}`, `prescrire{synthese}`,
   `reco_officielle{source,position,divergence:boolean,explication}`), `incertitudes[]`, `veille_liee[]`,
   `meta{date_revue,auteur,statut:'brouillon'|'valide',version,changelog[]}`.
2. `additionalProperties:false` aux bons niveaux, `required` cohérent avec le brief.
3. Types TS alignés 1:1 (source de vérité = le schéma ; les types le reflètent).

### Validation

- Auto : `npm run typecheck` OK. (La validation d'un nœud réel contre ce schéma est testée en T-004.)

### Si bloqué

- Ambiguïté de forme entre §5.1 et §11 (ex. `conditions` en tableau de chaînes) → suivre **§11** (gabarit
  concret) et documenter le choix en commentaire du schéma. Ne pas STOP pour un détail de forme.

## T-004 — Import YAML (plugin Vite) + validation Ajv en test

### Objectif

Rendre le contenu `/content/**/*.yaml` importable en JS au build, et vérifier chaque nœud contre le
JSON Schema dans un test Vitest.

### Lire / Modifier

- Modifier : `vite.config.ts` (ajouter le plugin YAML), `package.json` (devDeps). Créer :
  `src/features/decision/content/loadNodes.ts` (agrège les nœuds importés en `Noeud[]` typé),
  `src/features/decision/content/content.test.ts` (Ajv valide chaque nœud contre `schema/noeud.schema.json`).

### Étapes

1. `npm i -D @modyfi/vite-plugin-yaml ajv` ; enregistrer le plugin dans `vite.config.ts`. Déclaration de
   types pour l'import `*.yaml` si nécessaire (`src/vite-env.d.ts` ou `*.yaml.d.ts`).
2. `loadNodes.ts` : importer le(s) YAML de `content/noeuds/**` (via `import.meta.glob` ou import direct),
   typer en `Noeud[]`. Exposer un accès par `domaine` (préparer le multi-domaine : regrouper par `domaine`).
3. `content.test.ts` : charger le schéma, compiler avec Ajv, itérer sur tous les nœuds → `expect(valid)`.
