## T-001 — Scaffold Vite + React + TS + Vitest + tokens + shell

### Objectif

Créer l'app (Vite React-TS), la structure feature-first, les tokens OKLCH, et le shell applicatif
(header + navigation state-based + bandeau disclaimer) reproduisant la maquette.

### Décision clé

`DECISIONS.md` D9 : navigation **state-based** (un `screen` dans l'état, pas de routeur) ; **tokens =
variables CSS** OKLCH dans un `tokens.css` importé globalement ; styles co-localisés. Structure calquée
sur ETP interactif (générique dans `shared`, spécifique par module).

### Lire / Modifier

- Créer : `package.json`, `vite.config.ts`, `tsconfig*.json`, `index.html`, `src/main.tsx`, `src/App.tsx`,
  `src/styles/tokens.css`, `src/styles/global.css`, `src/features/shared/` (layout : `AppShell`,
  `Header`, `DisclaimerBar`, `AccountMenu`), `src/features/shared/navigation.ts` (type d'écran + état),
  `src/features/shared/types.ts` (types partagés : niveau de preuve, impact — pour réutilisation),
  `public/` (favicon neutre). Optionnel : `src/features/shared/badges/` (composants Badge preuve/impact
  génériques, réutilisés en Veille plus tard).

### Étapes

1. `npm create vite@latest . -- --template react-ts` dans le dossier (repo déjà init) ; nettoyer le
   boilerplate (logo, CSS de démo). Ajouter Vitest : `npm i -D vitest` ; script `"test": "vitest run"`,
   `"typecheck": "tsc --noEmit"`. Scripts `dev`/`build` (`tsc -b && vite build`)/`preview` standard.
2. `src/styles/tokens.css` : déclarer en `:root` les variables OKLCH du prototype (fond, surface,
   bordure, texte/atténué/faible, **accent-decision** `oklch(46% 0.09 254)`, **accent-veille**
   `oklch(58% 0.075 178)`, disclaimer, badges preuve/impact, rayons). Nommer clairement (`--c-surface`,
   `--c-accent-decision`, …). Importer `tokens.css` + `global.css` dans `main.tsx`.
3. `navigation.ts` : type `Screen = 'home' | 'decisionDomains' | 'decisionNode' | 'veilleList' |
   'veilleDetail' | 'profile' | 'memory' | 'auth' | 'methode'` + un hook/contexte d'état minimal
   (`useNavigation` → `{ screen, params, go(screen, params?) }`). Params = `{ nodeId?, articleId? }`.
4. `AppShell` : rend le header (logo « MSP Ménilmontant », pills Décision/Veille avec état actif, lien
   Méthode, menu compte stub) + `DisclaimerBar` quand `showChrome` (masqués sur `home` et `auth`,
   comme le prototype) + le contenu de l'écran courant. Header **sticky**, responsive (wrap mobile).
5. `App.tsx` : monte `AppShell` + un switch sur `screen` rendant pour l'instant **Home** (D1) et
   **Méthode** (S1) ; les autres écrans → placeholder neutre (« Bientôt » / retour accueil) sans les styliser.
6. Vérifier `npm run build` et `npm run typecheck` OK.

### Validation

- Auto (bloque le commit) : `npm run build` → succès (inclut typecheck) ; `npm run typecheck` → 0 erreur.
- Humain (visuel/UX, non bloquant) : header + pills + bandeau disclaimer fidèles au prototype ;
  responsive mobile (pas de débordement horizontal) → reporter dans `VALIDATION.md` (bloc « Shell »).

### Si bloqué

- Si `npm create vite` refuse d'écrire dans un dossier non vide (fichiers de contexte + `.git` présents) :
  scaffolder dans un sous-dossier temporaire puis déplacer `src/`, `index.html`, configs à la racine —
  **ne jamais** écraser les `.md` de contexte ni `docs/`, `design/`, `plans/`. En cas de doute → STOP.

### Message de commit (appliqué en fin de plan, cf. §4d)

`chore: scaffold Vite+React+TS app, OKLCH tokens, and app shell`
