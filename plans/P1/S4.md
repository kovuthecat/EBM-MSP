## T-006 — Écran D3 : nœud interrogeable (form → options → argumentaire)

### Objectif

Reproduire l'écran nœud : formulaire de critères → options applicables (badge preuve, effet, avantages/
inconvénients, « pourquoi ») via le moteur, puis argumentaire dépliable (reco officielle vs position
critique, drapeau de divergence, incertitudes, sources) + pied date de revue/disclaimer.

### Décision clé

Les options affichées viennent du **moteur S3** (`evaluateNode(node, criteria)`), recalculées à chaque
changement de critère (état éphémère, aucune persistance). La 1re applicable = « Recommandée » ; les
autres en alternatives. Le « pourquoi » vient des conditions satisfaites renvoyées par le moteur.
Nœud non détaillé (autres que cible, absents en P1) → bloc « Contenu détaillé à venir » du prototype.

### Lire / Modifier

- Créer : `src/features/decision/screens/DecisionNodeScreen.tsx`, sous-composants
  (`CriteriaForm`, `OptionCard`, `ArgumentPanel`) + styles. Réutiliser le Badge preuve de `shared`.

### Étapes

1. `CriteriaForm` : générer les champs depuis `criteres_entree` du nœud (nombre → input, enum → select,
   bool → checkbox), état local des critères (valeurs de départ raisonnables).
2. À chaque changement : `evaluateNode(node, criteria)` → liste d'options ordonnées ; rendre les
   `OptionCard` (intitulé, badge « Recommandée » sur la 1re, badge preuve coloré, effet attendu,
   avantages/inconvénients, ligne « Pourquoi cette option : … » depuis les conditions satisfaites).
3. `ArgumentPanel` (toggle) : deux colonnes reco officielle / position critique, **drapeau de divergence**
   si `reco_officielle.divergence`, incertitudes, sources (titre + type_critere). Pied : « Révisé le … ·
   Aide à la réflexion, le praticien reste responsable. »
4. Nœuds non détaillés → bloc placeholder. Build + typecheck OK.

### Validation

- Auto (bloque le commit) : `npm run build` + `npm run typecheck` OK ; les tests moteur (S3) restent verts.
- Humain : le filtrage réagit aux critères conformément au brief §11 (4 bandes) ; fidélité visuelle au
  prototype ; divergence/sources/incertitudes affichées ; mobile OK → `VALIDATION.md` (bloc « Décision D3 »).

### Si bloqué

- Écart entre le comportement attendu (brief §11) et ce que renvoie le moteur → **ne pas patcher l'UI**
  pour masquer : STOP, signale (le correctif est côté moteur S3 ou contenu S2).
